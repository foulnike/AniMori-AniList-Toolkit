# История изменений

Полная история версий также доступна на [Greasy Fork](https://greasyfork.org/ru/scripts/572948-animori-anilist-toolkit).

## [1.8.2]

Внутренняя модернизация: надёжность, диагностика и безопасность кода. Без изменений в видимой функциональности.

### Добавлено
- **Уровень логов `WARN`** — логгер теперь различает `INFO`/`WARN`/`ERROR`/`DB`/`API`/`QUEUE` с фильтрацией и своим цветом в модалке.
- **Глобальный перехват необработанных ошибок** (`window.onerror`, `unhandledrejection`) с фильтром по принадлежности скрипту, чтобы не засорять лог чужими ошибками страницы.
- **`safeCall()`** — единая обёртка для логирования ошибок в асинхронных вызовах.
- **Версионированные миграции IndexedDB** (`DB_MIGRATIONS`) — явный, расширяемый шаблон для будущих изменений схемы кэша вместо одного неявного блока.
- **`html\`\`` / `rawHTML()`** — безопасная сборка HTML-фрагментов с автоэкранированием интерполированных значений; переведены все места сборки интерфейса, куда попадают данные из Shikimori/AniList (защита от XSS).
- **JSDoc-типы** (`@typedef`) для структур данных AniList/Shikimori и записей кэша IndexedDB, плюс `@param`/`@returns` на ключевых функциях (сканер дельты, кэш, API-модуль) — даёт автодополнение и подсказки типов в редакторе без перехода на TypeScript.

### Изменено
- Расширено логирование в существующих `try/catch` блоках, ранее либо молчавших, либо логировавших недостаточно (экспорт списков, синхронизация избранного, запросы к Shikimori/AniList и др.) — упрощает диагностику багрепортов.

## [1.8.0]

Крупное обновление: сравнение списков, полный редизайн интерфейса и апгрейд плеера.

### Добавлено
- **Сравнение списков Shikimori ⇄ AniList** — новый виджет (кнопка «⇄»): сводная статистика по категориям прогресса, поимённые расхождения (статус, оценка, прогресс, пересмотры, заметки), сравнение избранного (аниме, манга, персонажи, персонал), детект «связанных» тайтлов (деление на сезоны), игнор-лист и глубокая проверка каталога батчами.
- **Музыкальные темы**: добавлены сервисы **Spotify** и **SoundCloud** (к VK и YouTube Music), брендовые иконки и свой акцентный цвет у каждого сервиса.
- **Плеер**: бесшовное переключение серий через Kodik Player API — без перезагрузки и перезапуска видео, не выходя из полноэкранного режима; автоподсветка текущей серии по событиям плеера.

### Изменено
- **Полный редизайн интерфейса** на едином UI-ките: стеклянные модалки с тема-нативными цветами (подстраиваются под тему и профиль-цвет AniList), прямоугольные переключатели. Переработаны настройки, экспорт, логгер и виджет сравнения.
- Кнопки действий (настройки, логгер, сравнение) объединены в единый блок; кнопка плеера прирастает к нему слева.
- Освежён дизайн виджетов на карточках: рейтинги-ярлыки, дерево франшизы (плюс закреплённая кнопка «Свернуть» сверху при разворачивании), музыкальные темы, внешние ссылки.
- Переработан оверлей плеера Kodik.
- Логгер включён по умолчанию.

### Исправлено
- MangaLib теперь ведёт на поиск по каталогу, а не на главную страницу.
- Цвета ярлыков оценок стабильны на страницах и аниме, и манги.
- Чипы внешних ссылок получают запасной цвет на темах, где часть токенов не определена.
- Устранена обрезка длинных списков в модалке сравнения (корректная прокрутка).
- Экспорт на Shikimori подстраивается под цвета темы сайта.
